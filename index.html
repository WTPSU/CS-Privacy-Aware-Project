Spreading Awareness for Local Differential Privacy (LDP)

By Westley T and Hong

Introduction

Differential privacy (DP) is a technique for preserving individual privacy. It does this by adding controllable noise to the data so that one person's data does not change the output significantly, making it harder for attackers to gain information.

Central differential privacy is a type of DP that is already widely used, its application is mostly on the server/host/creator side with raw data. For example, after raw data is collected from clients, DP is applied to ensure that the data is good and privacy-preserved for internal and external use.

Local differential privacy (LDP) on the other hand, is less common. LDP is applied on the client side before reaching the server, ensuring the data given is "accurate" enough to be useful while also preserving some privacy. For example, instead of the data being sent directly to the server, noise is added before it does, ensuring that individual privacy is preserved when it is used by a third party.

LDP could provide more privacy for everyone if it was adapted more. Since it is applied on the client before data is sent to servers, no raw data is collected by a 3rd party. When things like data-breaches happen, individual privacy is preserved, and it will be harder for attackers to isolate a single person's data. The trade-off for using LDP is accuracy, but it might be worth it if everyone's privacy is preserved.

The goal of this website is to spread awareness towards LDP with a detailed showcase with the hopes that it gets adapted more in practical scenarios, leading to more privacy for everyone overall.

The showcase source(s) can be found and ran at our repo on Github.
